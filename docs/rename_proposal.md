# プロジェクト名変更案

現在のプロジェクト名: **tazuna**
現在の由来: 「手綱」を握ってコントロールするイメージ。

## 新コンセプト
- **Lazy (怠惰)**: 開発者は極力働かない。
- **コーディングの完全自動運転**: 人間の介入を最小限にする自律型エージェント。

「手綱を握る」必要すらなく、完全に任せてしまえるものを目指すためのリネーミング案です。

## 提案候補：造語・固有名詞系

「Sloth（ナマケモノ）」の思想を継承しつつ、検索性やユニークさを高めた造語案です。

### 1. Zloth (ゾロス / ズロス)
- **由来**: **Zzz** (眠り) + **Sloth** (ナマケモノ)
- **コンセプト**: 開発者が寝ている(Zzz)間に、ナマケモノ(Sloth)のようなエージェントが仕事をする。
- **特徴**: 頭文字をSからZに変えることで、アルファベット順で最後（究極）感が出ると同時に、ハッカー文化的な響きを持つ造語になります。
- **ロゴイメージ**: 眠っているナマケモノ、あるいは稲妻(Zap)のような尻尾を持つナマケモノ。

### 2. Lazen (レイゼン)
- **由来**: **Lazy** (怠惰) + **Zen** (禅)
- **コンセプト**: 「何もしないこと」の境地。開発におけるノイズを削ぎ落とし、静寂の中でコードが生成されていく様。
- **特徴**: 短く、発音しやすく、洗練された印象を与えます。MavenやGradleのようなビルドツール系の響きもあり、開発ツールとして馴染みやすい。

### 3. Torpor (トーパ)
- **由来**: **Torpor** (動物の休眠状態)
- **コンセプト**: 動物（ユーザー）はエネルギーを節約するために休眠状態に入り、生命維持（開発）は自動化される。
- **特徴**: 生物学用語の転用。響きが可愛らしく（SherpaやDockerのような語感）、マスコットキャラクターも作りやすい。

### 4. Somna (ソムナ)
- **由来**: **Somnus** (ローマ神話の眠りの神) / **Somnolence** (眠気)
- **コンセプト**: 開発者を心地よい眠りへと誘うツール。起きたら世界（コード）ができあがっている。
- **特徴**: 神話由来の名称はKubernetesやApolloなど開発ツールで人気があり、権威性と優雅さを感じさせます。

### 5. Otium (オティウム)
- **由来**: ラテン語の **Otium** (公的な務めからの解放、学芸に耽る余暇)
- **コンセプト**: 単なる怠けではなく、「雑務（コーディングの実作業）から解放され、より創造的なことに時間を使う」という前向きなLazy。
- **特徴**: 知的でアカデミックな響き。「Negotium（仕事）」の対義語。

## 推奨
響きのユニークさと「眠りながら開発」というLazyさを体現する **Zloth**、あるいはミニマルでモダンな印象の **Lazen** を推奨します。
