# dursor OSS公開要件

## 概要

dursor を OSS として公開するための要件と計画をまとめたドキュメント。

### 背景

- 類似プロジェクト（AI coding agent）がパブリックリポジトリで増加
- dursor の差別化ポイントを活かしてOSS公開を決定
- 将来的な SaaS オプションも視野に入れる

---

## 1. ライセンス

### 決定事項

| 項目 | 内容 |
|------|------|
| ライセンス | **Apache License 2.0** |
| 変更前 | MIT License |
| 変更日 | 2026-01-16 |

### Apache 2.0 を選択した理由

1. **特許条項**：企業利用時の法的保護が明確
2. **貢献者保護**：コントリビューターからの特許ライセンスが自動付与
3. **企業フレンドリー**：多くのエンタープライズ企業が Apache 2.0 を好む
4. **SaaS 対応**：将来の SaaS 展開時にも柔軟性がある

## 5. 将来の SaaS オプション

### Open Core モデル設計

```
dursor (Apache 2.0 - OSS)
├── Core機能（無料）
│   ├── マルチモデル並列実行
│   ├── セルフホスト
│   ├── 基本 UI
│   └── GitHub PR 連携
│
└── dursor Cloud（有料 SaaS）- 将来検討
    ├── マネージドホスティング
    ├── チーム管理機能
    ├── SSO/SAML 認証
    ├── 監査ログ
    ├── SLA 保証
    └── 優先サポート
```

### SaaS 検討トリガー

以下の条件が揃った場合に SaaS 版を検討：

1. Stars 3,000+ 達成
2. 「ホスティング版がほしい」という要望が複数
3. 開発リソースの確保
4. 運用コスト（インフラ、サポート）の見通し

## 7. 成功確率を上げる施策

| 施策 | 効果 | 難易度 | 優先度 |
|------|------|--------|--------|
| 英語でのプロモーション | 高 | 中 | **高** |
| Hacker News 投稿 | 高（当たれば） | 低 | **高** |
| YouTube デモ動画 | 中 | 中 | 中 |
| Twitter/X での継続発信 | 中 | 低 | 中 |
| 競合との明確な差別化 | 高 | 高 | **高** |
| 有名人・インフルエンサーの推薦 | 高 | 高（運） | 低 |

## 8. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| 競合の台頭 | Star 獲得困難 | 差別化機能に集中、ニッチを狙う |
| メンテナンス負担 | バーンアウト | 期待値を下げる、無理をしない |
| セキュリティ問題 | 信頼低下 | Dependabot 活用、迅速な対応 |
| スポンサー特典の履行 | 信頼低下 | 無理のない特典設計 |
